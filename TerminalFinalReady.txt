Reason for the change
npm 11 (bundled with Node 24.16.0) removed support for `msvs_version` as a valid npm config key. The old `npm config set msvs_version 2022 --location=project` line caused "npm error `msvs_version` is not a valid npm option". The fix removes the invalid command while preserving the critical `--arch=x64` flag required for node-pty on Electron 42 Windows x64 builds.

Confirmation that TerminalBuildFix4-SpectreAndNpm11.txt changes are now applied
Confirmed. Inspection of package.json scripts section shows postinstall and rebuild now contain only the clean `electron-rebuild --force --arch=x64` commands exactly as specified in TerminalBuildFix4-SpectreAndNpm11.txt. The invalid msvs_version line has been removed.
